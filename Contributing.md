# Contributing to TokaLab

TokaLab is an open-access, open-source project, and we warmly welcome contributions from the community.
Whether you are a researcher, student, or developer working in nuclear fusion and plasma physics, there are several ways to get involved.

---

## 🛠 Ways to Contribute

### Report a bug or suggest a feature
Open an [Issue](https://github.com/TokaLab/VirtualLab/issues) on GitHub.
Please provide a clear description, steps to reproduce (for bugs), and the relevant module and language (MATLAB or Python).

### Fix a bug or improve existing code
1. Fork the repository.
2. Create a branch with a descriptive name (e.g., `fix/syndiag-bug` or `improve/simpla-docs`).
3. Make your changes and test them.
4. Open a Pull Request with a clear description of what you changed and why.

### Contribute a new module
If you have developed a tool or algorithm relevant to VirtualLab, you can transfer it to TokaLab:
1. Contact the team at [tokalab.fusion@gmail.com](mailto:tokalab.fusion@gmail.com) to discuss your module before starting.
2. Follow the steps above (fork → branch → Pull Request).
3. Include a `README.md` in your module folder describing its purpose, inputs/outputs, and usage.
4. The admin team will review the submission and provide feedback.

Once accepted, your module will be published and **you will be credited** in the [Contributors](./Contributors.md) file.
If your module is based on published work, you may include a citation request in your module's `README.md`.

---

## 📐 Code Guidelines

There are no strict style requirements, but please keep the following in mind:

- Write **clear, readable code** with comments where needed.
- Include at least one **minimal working example** in your module.
- Use **descriptive variable names** — avoid unexplained single-letter variables.
- If your module has dependencies, document them explicitly.
- MATLAB and Python implementations should follow the existing folder structure (see the main [README](./README.md)).

## 👥 Joining the TokaLab Team

If you are interested in becoming a TokaLab team member and participating in meetings and project activities, reach out to us at [tokalab.fusion@gmail.com](mailto:tokalab.fusion@gmail.com).
